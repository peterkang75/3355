### Phase 7: 라운딩 사진·동영상 갤러리 ⬜

> 저장소: **Cloudflare R2** (무료 10GB, 조회 트래픽 무료). 앱 DB에는 R2 객체 key + 메타데이터만 저장.
> 이미지 서빙: **서명 URL(signed URL)** 방식 — 도메인 구입 불필요, R2 무료 트래픽 유지.

**확정 설계:**
- 사진 + 짧은 동영상 모두 지원. 동영상은 서버에서 자동 압축(ffmpeg, 720p) 후 저장.
- 업로드/삭제 권한: 그 라운딩 **참가 회원만**. 개별 사진 삭제는 **올린 사람만**. 월별 대량 정리는 **관리자만**.
- 지난 라운딩 진입: 라운딩 라운지 상단 **예정/지난 토글** → 지난 선택 시 월별 역순 목록(📷 개수 뱃지).
- 라운딩 상세: **정보 / 스코어 / 사진** 3탭.
- 용량 관리: 관리자 "저장소 관리" 화면 — 사용량 막대 + 월별 zip 다운로드(즉석 생성, 미리 저장 안 함) → 삭제.
- 정리된 라운딩: `Booking.photosArchivedAt` 플래그 → 사진 탭에 **"백업 후 정리됨"** 안내.

| Step | 작업 | 상태 |
|------|------|------|
| 1 | Cloudflare R2 가입 + 버킷 생성 + API 토큰 → 로컬 .env + Railway 환경변수 연결 | ✅ 2026-05-25 (버킷 golf-3355-media, 서명URL 방식) |
| 1.5 | 미디어 도구 점검: R2 연결 + ffmpeg-static 스모크 테스트 통과. 썸네일은 기존 jimp 재사용 | ✅ 2026-05-25 |
| 2 | RoundingMedia 테이블 + 업로드/조회/삭제 API + 영상압축·썸네일(jimp/ffmpeg) + 참가자 권한 검증 | ✅ 2026-05-25 (파이프라인 e2e 테스트 통과) |
| 3 | 라운딩 라운지 예정/지난 토글 + 지난 라운딩 월별 역순 목록 (📷 개수 뱃지) | ✅ 2026-05-25 |
| 4 | 라운딩 상세 3탭 + 갤러리 UI (그리드 → 확대 → 다운로드 → 단일 삭제) | ⬜ |
| 5 | 저장소 관리(관리자): 사용량 막대 + 월별 다운로드(zip)/삭제 + "백업 후 정리됨" 안내 | ⬜ |
| 6 | 업로드 제한 (사진 2MB·영상 원본 200MB/압축후 ~30MB, 라운딩당 30개·500MB) + (선택) 80% 텔레그램 알림 | ⬜ |

**기술 결정/주의 (역검수):**
- 이미지 서빙 = 서명 URL (도메인 불필요, 갤러리 로드 시 서버가 단기 URL 발급).
- 영상 압축은 시간 소요 → 업로드는 "처리 중" 비동기 상태로, 모바일 타임아웃 회피.
- HEIC(아이폰 기본) → JPEG 변환 필수, EXIF 회전 보정.
- "백업 후 정리됨" = (b) RoundingMedia 행 삭제 + Booking.photosArchivedAt 세팅 방식.

**완료 기준:**
