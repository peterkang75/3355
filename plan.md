### 회비 차등 청구 — 골프장 멤버 항목별 면제 ✅ (2026-06-05)

> 배경: 6월 정기모임이 스트라스필드 골프클럽 — 클럽 소속 회원은 그린피 면제, 카트비·참가비는 동일 청구. 기존 라운딩은 fee 3개를 전원에게 동일 청구.

**확정 설계:**
- 라운딩의 비용 안내 섹션에 각 항목별 "클럽 멤버 면제" 토글 3개 (그린피/카트비/참가비)
- 토글 ON 시 `member.club === booking.courseName`인 회원에게는 해당 항목 청구액 0
- 비-멤버·club 미입력 회원은 일반 금액 그대로
- description에 면제 사유 자동 메모: "(클럽 멤버: 그린피 면제)" 등
- 모달에 면제 대상 회원 명단 미리보기(N명) — 0명이면 경고로 표시(데이터 정비 유도)

| Step | 작업 | 상태 |
|------|------|------|
| 1 | `Booking.waiveGreenFee/CartFee/MembershipFeeForClubMembers` Boolean 3컬럼 추가 + 운영 DB push | ✅ 2026-06-05 |
| 2 | `bookings.js` — 회원별 청구 계산 헬퍼(`computeMemberChargeForBooking`)로 통합, 참가자 추가·toggle-join·fee 동기화 3곳 일관 처리. waive 토글 변경도 동기화 트리거에 포함 | ✅ 2026-06-05 |
| 3 | `CreateBookingModal`·`HostManageSheet` 비용 섹션에 미니 토글 3개 + 면제 대상 명단 미리보기. 골프장 미선택 시 토글 비활성 | ✅ 2026-06-05 |
| 4 | 빌드(vite)·서버 부팅·문법 검증·props 전달 점검. 실 시나리오(라운딩 생성 후 멤버/비-멤버 참가)는 운영 잔액 영향 회피 위해 사장님 6월 라운딩 생성 시 확인 | ✅ 2026-06-05 |

**구현 메모:**
- 자동 분류 기준은 문자열 완전 일치(`member.club === booking.courseName`). 회원 정보의 소속 클럽 필드가 정확히 입력돼야 함. 현재 운영 DB Strathfield 소속 15명 등록 확인.
- 미래 확장(회원 직접 선택 등 다른 면제 조건)은 후속 — 현재는 "해당 골프장 멤버" 조건 단일.
- 게스트 charge(`category: '게스트 참가비'`)는 회원 면제 규칙 미적용 — 동기화 루프에서 스킵.
- 이미 납부된 charge(payment 트랜잭션 존재)는 fee 동기화에서 스킵 — 기존 안전 로직 유지.

### Phase 9: 소식 커뮤니티 피드 ✅ (2026-05-27)

> 목표: 지난 라운딩 사진을 한 곳(소식 탭)에 모아 인스타식으로 보고, 반응·댓글로 소통. 사진 동선 강화로 커뮤니티 활성화.

**구현 메모:**

- 피드 = **라운딩 자동 게시물**(사진 있는 지난 라운딩, 라운딩당 1개, 제목 자동) + **자유 사진글**(정회원) 혼합, 최신순(새 사진 올리면 위로).
- **업로드 입구 통일**: 우하단 + → "정기모임 사진"(내가 참가한 지난 라운딩 선택 → 기존 라운딩 미디어 업로드 재사용) / "자유 게시물"(글+사진).
- 반응(♡)·댓글: 공용 **다형성 Reaction/Comment 테이블**(targetType booking|feedpost). 게스트는 보기·반응만, 댓글·작성은 정회원. 댓글 삭제는 작성자/운영자.

- 링크: 유튜브 임베드 / 인스타·일반은 링크 카드(서버 OG 미사용).
- 신규 테이블 4개(`FeedPost`/`FeedPostMedia`/`Reaction`/`Comment`) — 운영 `prisma db push`로 추가(기존 데이터 무손상, 컬럼 변경 0). `media.js`(라운딩 업로드) 무수정, 압축 프리미티브만 `feedMedia.js`로 재사용.
- 역검수 수정: 과거 판정 **시드니 시간대** 문자열 비교, 반응/댓글 대상 존재검증(404), 좋아요 더블탭 경합 무시, 라운딩 삭제 시 연결 반응/댓글 정리, 댓글 카운트 실시간 갱신.

### Phase 7: 라운딩 사진·동영상 갤러리 ⬜

> 저장소: **Cloudflare R2** (무료 10GB, 조회 트래픽 무료). 앱 DB에는 R2 객체 key + 메타데이터만 저장.
> 이미지 서빙: **서명 URL(signed URL)** 방식 — 도메인 구입 불필요, R2 무료 트래픽 유지.

**확정 설계:**
- 사진 + 짧은 동영상 모두 지원. 동영상은 서버에서 자동 압축(ffmpeg, 720p) 후 저장.
- 업로드/삭제 권한: 그 라운딩 **참가 회원만**. 개별 사진 삭제는 **올린 사람만**. 월별 대량 정리는 **관리자만**.
- 지난 라운딩 진입: 라운딩 라운지 상단 **예정/지난 토글** → 지난 선택 시 월별 역순 목록(📷 개수 뱃지).
- 라운딩 상세: **정보 / 스코어 / 사진** 3탭.
- 용량 관리: 관리자 "저장소 관리" 화면 — 사용량 막대 + 월별 zip 다운로드(즉석 생성, 미리 저장 안 함) → 삭제.
- 정리된 라운딩: `Booking.photosArchivedAt` 플래그 → 사진 탭에 **"백업 후 정리됨"** 안내.
