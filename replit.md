# 골프 모임 관리 앱

## Overview
이 프로젝트는 한국어 모바일 웹 기반 골프 모임 관리 애플리케이션입니다. 회원들이 골프 모임을 효율적으로 관리할 수 있도록 부킹(라운딩), 스코어, 참가비 등을 통합 관리하는 것을 목표로 합니다. 주요 기능으로는 간편 로그인, 개인화된 대시보드, 게시판, 라운딩 관리(생성, 참가, 조편성, 스코어 입력), 핸디캡 자동 계산, 참가비 관리, 그리고 다양한 관리자 기능이 있습니다. 궁극적으로 골프 모임 운영의 효율성을 높이고 회원들에게 편리한 경험을 제공하는 것이 비전입니다.

## User Preferences
- 나는 간결하고 명확한 설명을 선호한다.
- 개발 과정은 반복적으로 진행하고 싶다.
- 주요 변경사항을 적용하기 전에 항상 내 의견을 물어봐야 한다.
- 코드 변경 시에는 충분한 설명을 제공해주면 좋겠다.
- 폴더 `Z`와 파일 `Y`는 수정하지 않는다.

## System Architecture

### UI/UX Decisions
- **모바일 최적화 및 반응형 레이아웃**: 최대 너비 600px로 모바일 환경에 최적화되어 있으며, 다양한 기기에서 잘 보이도록 반응형 디자인 시스템이 적용되어 있습니다.
- **색상 테마**: 골프 테마를 반영한 초록색 계열(`%2d5f3f`, `%3a7d54`, `%4a9d6a`)을 사용합니다. 배경색은 라이트 그레이(`#F3F4F6`), 카드는 화이트(`#FFFFFF`), 보조 배경은 `#F8FAFC`.
- **대시보드**: 개인화된 정보(핸디캡, 참가비 잔액, 다가오는 라운딩, 최근 스코어)를 표시합니다. 날짜 형식은 MM/DD HH:mm, 스탯 카드는 아이콘(⛳, 💳)과 큰 숫자(28px, 800 weight)로 표시됩니다.
- **라운딩 UI**: 현재 활성 라운딩과 완료된 라운딩을 구분하여 리스트 형태로 제공하며, 공지된 라운딩은 대시보드에 카드 형태로 표시됩니다.
- **조편성 UI**: 드래그 앤 드롭 기능을 통해 참가자들을 팀으로 편성할 수 있으며, 모바일 터치를 완벽하게 지원합니다.
- **관리자 메뉴**: 탭 대신 카드 형태의 서브메뉴 리스트 방식을 채택하여 명확성을 높였습니다.
- **참가비 메뉴**: 개인 참가비 내역과 모임 참가비 내역을 탭으로 구분하여 제공합니다.
- **금액 표시**: 모든 금액은 `$1,234` 형식으로 표시됩니다.

### Technical Implementations
- **로그인 시스템**: 전화번호 끝 6자리 기반의 간편 로그인 및 데이터베이스 기반 회원 인증을 사용합니다.
- **권한 시스템**: Role 기반 권한 관리로 관리자, 방장, 운영진, 클럽운영진, 회원 역할을 구분합니다. 게시판 작성 및 라운딩 관리는 운영진 이상의 역할에 부여되며, 참가비관리 권한은 개별적으로 설정 가능합니다. 앱 설정은 관리자 전용입니다.
- **라운딩 관리**: 운영진 이상이 라운딩을 생성하고, 골프장 선택, 비용 정보, 접수 마감일, 회식 정보 등을 상세히 설정할 수 있습니다. 참가자 관리 및 공지 활성화/비활성화 기능이 포함됩니다. 라운딩별 플레이 활성화 상태를 수동 및 자동으로 관리할 수 있습니다. 외부 게스트 추가 기능이 지원됩니다.
- **스코어 입력**: 홀별 타수 입력, 실시간 포인트 계산, 라운드 시간 측정 기능을 제공하며, 저장된 홀별 PAR 데이터를 활용할 예정입니다. 스코어는 플레이 중 localStorage와 서버에 실시간으로 자동 저장됩니다 (500ms 디바운싱). 리더보드 화면 전환 후 돌아와도 데이터가 유지됩니다. **스코어 삭제 기능은 완전히 비활성화**되어 있으며, 서버 API DELETE 엔드포인트도 403을 반환합니다. 앱 재시작 시 localStorage → 서버 DB 순서로 3단계 복원을 수행합니다 (localStorage 복원 → 서버에서 기존 스코어 불러오기 → 초기화).
- **Net 2-Ball Best Ball (2BB)**: 라운딩 설정에서 2BB 모드를 활성화하면 핸디캡 순위에 따라 자동으로 팀이 편성됩니다 (4인: 1위+4위 팀A, 2위+3위 팀B / 3인: 1위+2위 팀A, 1위+3위 팀B - 피벗). 마커 선택 시 2BB 파트너에게 주황색 배지가 표시되며, 4인 조에서는 파트너를 마크할 수 없습니다. 리더보드에서 각 홀의 더 낮은 Net Score를 선택하여 팀 랭킹을 계산합니다.
- **핸디캡 계산**: 스코어 데이터를 기반으로 하우스 핸디를 자동 계산하며 최근 라운드 기록을 분석하고 평균 스코어를 추적합니다.
- **참가비 관리**: 다양한 참가비 항목 생성, 납부 내역 추적, 미수금 자동 계산 및 지출 처리 기능을 포함합니다. `canManageFees` 권한을 가진 회원만 참가비 관리 가능합니다. 개인 참가비 내역과 모임 참가비 내역(통합 장부)을 탭으로 구분하여 조회할 수 있으며, 통합장부에서 청구 내역 필터링이 가능합니다. 회비 면제 기능과 회원 크레딧 기능(지급, 사용, 기부, 미납금 납부)이 통합되어 있습니다.
- **관리자 기능**: 회원, 권한, 참가비, 골프장, 지출 항목을 관리할 수 있습니다. 앱 설정 메뉴는 관리자 전용입니다. 입금/출금 항목 관리 기능이 포함됩니다.
- **상태 동기화**: `AppContext`를 통한 전역 상태 관리 및 변경 후 자동 새로고침으로 데이터 일관성을 유지합니다. Socket.IO를 통해 빙고 게임 설정, 라운딩 활성화 상태 등 실시간 UI 업데이트를 지원합니다. Socket.IO 이벤트에 300ms 디바운싱이 적용되어 빠른 연속 호출 시 API 요청을 최적화합니다.
- **Race Condition 방지**: 서버 측 토글 로직을 사용하여 동시성 문제를 해결합니다.
- **잔액 증분 업데이트**: 거래 생성/수정/삭제 시 전체 거래를 재계산하지 않고 Prisma의 increment/decrement를 사용하여 잔액을 증분 업데이트합니다. '크레딧 자동 납부'와 '크레딧 납부' 카테고리의 payment는 잔액에 영향을 주지 않습니다.
- **검색 기능**: 모든 골프장 선택 드롭다운에 검색 기능이 추가되었습니다.

### Feature Specifications
- **로그인**: 전화번호 끝 6자리 인증
- **대시보드**: 핸디캡, 참가비 잔액, 부킹 일정, 최근 스코어
- **게시판**: 공지사항, 댓글, 실시간 알림, 운영진 이상 작성 권한
- **라운딩**: 라운딩 생성/수정/삭제, 참가/취소, 조편성 (드래그 앤 드롭, 자동 배정), 스코어 입력, 공지 기능, 플레이 활성화 제어, 외부 게스트 추가
- **스코어**: 홀별 타수 입력, PAR, SHOTS, PICK UP, TOTAL 표시, 실시간 포인트 계산
- **핸디캡**: 하우스 핸디 자동 계산, 최근 기록 분석, GA/HH 형식 표시
- **참가비**: 항목 생성, 납부 추적, 미수금 계산, 지출 처리, 개인/모임 참가비 내역 탭 조회, 회비 면제, 회원 크레딧 관리 (지급, 사용, 기부, 미납금 납부), 지출 거래 카테고리/메모 분리, 청구 내역 필터링
- **관리자**: 회원, 권한, 참가비, 골프장, 지출 항목 관리, 입금/출금 항목 관리
- **빙고 게임**: 운영자 설정 관리, 회원별 개인 빙고판, 실시간 동기화
- **사용자 활동 추적**: 페이지 방문 로그, 현재 접속중인 회원 표시, 7일 자동 정리

### Code Architecture
- **컴포넌트 분리**: 대형 페이지 컴포넌트를 재사용 가능한 하위 컴포넌트로 분리하여 유지보수성을 높였습니다.
  - `src/components/common/PageHeader.jsx`: 페이지 헤더 컴포넌트 (뒤로가기 버튼, 제목, 우측 컨텐츠). Props: `title`, `onBack`, `rightContent`, `showBackButton`, `variant`
  - `src/components/common/ProfileBadge.jsx`: 프로필 배지 컴포넌트 (닉네임, 프로필 사진)
  - `src/components/booking/BookingForm.jsx`: 라운딩 생성/수정 폼 컴포넌트
  - `src/components/booking/BookingListCard.jsx`: 라운딩 목록 카드 컴포넌트 (활성/완료 라운딩)
- **PageHeader 적용 페이지**: Dashboard, Booking, MyPage, Fees, RoundingManagement, TeamFormation, ParticipantManagement, MemberScoreEntry - 서브 페이지는 해당 페이지 컨텍스트에 맞는 제목 표시 (예: "라운딩 관리", "조편성", "참가자 관리", "스코어 입력")
- **ROOT_PATHS**: `['/', '/booking', '/fees', '/mypage', '/admin']` - 이 경로에서는 뒤로가기 버튼 숨김
- **스코어 계산 수정**: Over/Under 계산 시 실제로 플레이한 홀의 PAR만 합산하도록 수정 (스코어가 0보다 큰 홀만 계산)
- **포썸 모드 표시**: 라운딩 카드에 포썸 경기 방식을 보라색 배지로 표시
- **CSS-First Design System**: JSX 구조를 변경하지 않고 `globals.css`에서 전역 스타일을 정의하여 디자인 시스템 적용
  - `.card` 클래스: 흰색 배경, border-radius 16px, 플로팅 효과 (box-shadow)
  - 버튼 클래스: `.btn-primary`, `.btn-outline`, `.btn-danger`, `.btn-ghost`
  - 게임 페이지 (Play, Leaderboard): 다크 배경 + 흰색 플로팅 카드 조합

### System Design Choices
- **프론트엔드**: React 19, Vite 7을 사용하여 모던하고 빠른 웹 애플리케이션을 구축합니다. React Router DOM으로 클라이언트 측 라우팅을 관리합니다.
- **백엔드**: Node.js와 Express를 사용하여 RESTful API를 제공합니다.
- **데이터베이스**: PostgreSQL을 사용하며, Neon-hosted 환경에서 운영됩니다.
- **ORM**: Prisma를 사용하여 데이터베이스 스키마 관리 및 쿼리를 효율적으로 처리합니다. Prisma의 `cuid()`를 사용하여 자동 ID를 생성합니다.
- **개발 환경**: Concurrently를 사용하여 프론트엔드와 백엔드 개발 서버를 동시에 실행합니다.
- **프로덕션 환경**: Express가 빌드된 정적 파일과 API를 함께 제공합니다.

## External Dependencies

- **Database**: PostgreSQL (Neon-hosted)
- **ORM**: Prisma (PostgreSQL과의 상호작용을 위한)
- **Real-time Communication**: Socket.IO
- **Image Compression (Frontend)**: browser-image-compression (max 400px, 0.2MB, JPEG) — `src/utils/compressImage.js`
- **Image Compression (Batch Script)**: jimp — `server/scripts/compress-photos.js` (`npm run compress:photos`)
